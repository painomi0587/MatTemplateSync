## 目的

v0.1（D&D + カテゴリ選択 + 一括適用 + Undo）を Unity 2022.3 + lilToon 1.7〜1.8 実機で検証し、
コンパイルエラー・プロパティ名の綴り齟齬・UI 不具合を修正して動作する状態にする。

## 現状の挙動

コードはレビュー済み・コミット済みだが、Unity 実機でのコンパイル・動作確認が一度も行われていない。
プロパティ名テーブルは lilToon 1.8.0 の lts.shader から採取したが、実マテリアルでの適用結果は未検証。

## 変更内容（項目）

1. **コンパイル確認**（オーナー実機）: git URL でパッケージをインストールし、コンパイルエラーを解消する
   - 特に `Material.SetInteger/GetInteger`, `ShaderPropertyType.Int`, `EditorStyles.linkLabel`, `AssetPreview.IsLoadingAssetPreview` の存在確認
2. **適用動作の確認**: 影/リム/ライティング設定カテゴリを2〜3マテリアルへ適用し、以下を確認する
   - 適用件数・スキップ件数のレポートが実プロパティ数と整合するか（タイポはサイレントスキップになるため件数で検知する）
   - Ctrl+Z 1回で全対象が復元されるか
   - バリアント違い（無印 ↔ Cutout 等)の組で安全にスキップされるか
3. **要実機確認事項の消化**（`Documentation~/DESIGN.md` §7）
   - シェーダー設定（`LIL_FEATURE_*`）でコンパイル除外中の機能をトグル 0→1 にしたときの挙動
   - Material Variant への適用でオーバーライドが増える点の UX
4. 発見した不具合の修正とコミット
