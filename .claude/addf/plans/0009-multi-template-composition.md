# Plan 0009: マルチテンプレート合成（検討スタブ）

## 実装状況: 未着手

> 出典: 2026-07-08 マルチエージェントリファクタリング検討（新機能提案エージェントの提案4）

## 分かっていること

- 「影は マテリアルA、リムは マテリアルB」のようにカテゴリスロットごとに異なるテンプレートを
  割り当てて1回で適用したい実需がある（複数アバター間の質感ミックス）。現状は2回に分けて適用し
  Undo も2回に分かれる
- 実装の骨子: `TemplateApplier` に
  `ApplyToMaterials(IReadOnlyList<(Material template, SyncCategory mask)> layers, targets)`
  オーバーロードを追加し、**Undo グループを外側で1つに括る**。単体適用は既存 `ApplyToMaterial` を
  レイヤ数だけ回すだけで純関数構造は不変
- 現行実装は呼び出しごとに `IncrementCurrentGroup` するため、グループ括り出しのリファクタが前提
  （[Plan 0004: Undo トランザクション統合](0004-undo-transaction-hardening.md) で担い手を
  Applier に集約した後なら差分が小さい）
- [Plan 0006: スタイルプリセットライブラリ](0006-style-preset-library.md) 実装済みなら
  「プリセットの重ね掛け」として自然に実現でき、実装コストが下がる
- UI はテンプレート欄を「既定テンプレート + カテゴリ別オーバーライド」に拡張し、
  デフォルト非表示のアドバンスド枠に隔離する
- テンプレート間でバリアントが異なる場合の警告は Plan 0002 のバリアント警告をテンプレート数分に拡張

## 未解決の問い

- カテゴリ重複時（同一カテゴリに複数テンプレート）の優先順位規則
- UI の複雑化に対する需要の強さ（オーナーの実運用頻度）
